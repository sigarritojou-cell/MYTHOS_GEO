# AGENTS

Este proyecto incluye una pequeña cuadrilla de agentes pensada para mantener el repositorio limpio, coherente y publicable sin mezclar decisiones de arquitectura, poda de dependencias y publicación.

## Orden recomendado

1. `agents/01_repo_auditor.md`
2. `agents/02_dependency_pruner.md`
3. `agents/03_structure_curator.md`
4. `agents/04_qa_gatekeeper.md`
5. `agents/05_repo_publisher.md`

## Regla principal

Cada agente modifica solo su ámbito. Nada de tocar veinte frentes a la vez y dejar el proyecto hecho un solar.
